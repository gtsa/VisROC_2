Input

  • P: number of positive events

  • Q: number of negative events

  • Resolution: number of segments in which the interval [0,1] is divided for the calculation of the p-values on a square lattice in the ROC diagram

  • User defined AUC: value of the Area Under the Curve (AUC) of a ROC diagram,for which you would like to calculate the p-value

  • F1: the horizontal coordinate F1 (depending on the view mode either the False Alarm Rate or the 1-Specificity value or the False Positive Rate) of the point (F1, H1) on the ROC diagram through which passes the k-ellipse for which you want to calculate the p-value
    Attention: F1 < H1

  • H1: the vertical coordinate H1 (depending on the view mode Hit Rate or the Sensitivity or the True Positive Rate) of the point (F1, H1) on the ROC diagram through which passes the k-ellipse for which you want to calculate the p-value
    Attention: H1 > F1

  • Data input file: So as for the data input file to be compatible with VISROC_2.0.exe (regardless of file type), its content should be organised in 2 columns. Otherwise, the calculations and plots of the program will be carried out without taking into account the ROC data entered by the user

Files

The output files (and their default names and types) that the user can save are:

  • out_field.csv: 3-column .csv file containing: the coordinates of the points in the ROC diagram for which calculation has been made together (cols x & y) and the corresponding p-values (col z)

  • out_CL.csv: 4-column .csv file containing as first column the F coordinate of the points of all three k-ellipses (10%, 5% and 1%), the H coordinates of which are given in the second, third and fourth column, respectively

  • out_p.csv: one line .csv file containing

    1. the p-value for the user-defined Area Under the Curve value (AUC0),

    2. the user-defined Area Under the Curve value (AUC0),

    3. the user-defined number of positive events P,

    4. the user-defined number of negative events Q and

    5. an error code i-fault which should be 0, 1 or 2 (see "Faults").

  • out_F1H1.csv: one line .csv file containing:

    1. the p-value of the k-ellipse passing through the point with coordinates (F1,H1),

    2. the value of F1,

    3. the value of H1,

    4. the value of i-fault

    5. the number of positive events P,

    6. the number of negative events Q and

    7. the AUC of the k-ellipse.

  • out_k_F1H1.csv: 3-column .csv file containing the coordinates of the k-ellipse passing from the point (F1, H1) and the corresponding value of k

Faults (i-fault codes)

  • No fault (i-fault = 0)

Back to Table of Contents

  • Non-Critical fault (i-fault = 1, corresponding input value in red font color): Some illegal input value(s); plots and calculations were nevertheless executed using altered values instead of the illegal ones.

    Possible causes:

    • User-defined AUC has a numerical value in the interval (0, 0.5), i.e. out of the closed interval [0.5, 1] (use of default value)

    • F1 or/and H1 have a numerical value out of the closed interval [0, 1] (use of default value)

    • H1 is smaller than F1 (invert values)

    • Filepath does not correspond to file (calculate and plot without user's ROC data input)

    • File format is not supported (calculate and plot without user's ROC data input)

    • Data dimensions are not correct, input data has not 2 cols (calculate and plot without user's ROC data input)

Back to Table of Contents

  • Critical fault (i-fault = 2, corresponding input value in red background color): Some illegal input value(s); plotting and calculations are not possible.

    Possible causes:

    • User-defined AUC has a numerical value in the interval below 0 or greater than 1 (AUC can only be in the closed interval [0.5, 1])

    • P or/and Q are not positive integers (P and Q can only be positive integers)
